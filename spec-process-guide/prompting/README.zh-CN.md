## 关键原则

用于规范开发的有效提示遵循以下原则：

1. **具体明确** - 提供清晰的上下文和具体示例
2. **结构化请求** - 将复杂请求分解为可管理的部分
3. **深思熟虑地迭代** - 在先前回应的基础上进行构建，而不是重新开始
4. **验证理解** - 在进入下一阶段之前确认一致性

## 常见模式

- **设置上下文** - 建立项目背景和约束
- **阶段转换** - 在需求、设计和任务之间平稳过渡
- **反馈整合** - 有效地整合变更和改进
- **质量验证** - 确保输出符合您的标准
